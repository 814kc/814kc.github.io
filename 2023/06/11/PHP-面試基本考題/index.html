  1. POST和GET有什麼差異？

    GET請求通常用於獲取或檢索資源，而POST請求通常用於向服務器提交或創建新的數據。

    其差異包含下列項目：

    • 請求方式：GET通過URL發送數據，POST的通過http請求體發送數據。
    • 安全性：GET請求參數明文顯示於URL，因此安全性較低，POST請求參數在http請求體中，相對安全。
    • 緩存：GET可被緩存，POST不可被緩存。
    • 數據長度：GET請求數據長度有限制通常在2048字符內，POST數據長度沒有限制。

  2. 什麼是Web Service？
    Web Service（網路服務）是一種基於 Web 技術，可以讓應用程式之間進行互動通訊的機制。它們可以是一種標準化的方式，讓應用程式使用網際網路上的遠端服務，並且跨平台、跨程式語言使用。
    Web Service 的主要目的是讓應用程式之間進行互動，並能夠透過網際網路來擴展和提供其功能。Web Service 通常使用標準化的網路協定和格式，如 HTTP、XML 和 SOAP，來實現應用程式之間的通訊。這些標準化協定和格式的使用，使得不同平台和程式語言之間的互操作性變得更加容易。
    Web Service 可以提供多種不同的功能，例如：

    • 查詢資料庫
    • 存取檔案
    • 執行遠端程式
    • 註冊會員
    • 獲取資訊
      Web Service 的使用者可以使用基於 Web 的應用程式或其他 Web Service 來呼叫這些服務。這些服務可以是由企業、政府或其他組織提供，也可以是由個人開發和提供的。

  5. include 和 require 之間的差異？
    include: 這個函式一般是放在流程控制的處理區段中，include在引入檔案時，如果碰到錯誤，會給出提示，但會繼續運行之後的程式碼。
    require: 通常放在 PHP 程式的最前面，PHP 程式在執行前，就會先讀入 require引入的檔案，require在引入檔案時，如果碰到錯誤，會給出提示，但中止運行。
